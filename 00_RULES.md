## 第 0 章：研究使命与基本原则

### 0.1 研究目的

通过对超导研究的系统化、长期、深度积累，建立**全局性、深刻性**的物理认识。

**核心原则：**
- ✅ **不为快速发文** — 任何子课题完成不要求最终产出论文
- ✅ **质量大于数量** — 一篇深度精读优于五篇浅读
- ✅ **诚实与可追溯** — 所有论断必须有源
- ✅ **长期主义** — 这是 6-18 月的长跑，不是冲刺

### 0.2 三层认知层级（强制标注）

每个论断必须明确标注层级：

| 层级 | 含义 | 标注示例 |
|------|------|---------|
| **[定理]** | 已严格证明 | "[定理] BCS 给出 2Δ/kBTc = 3.528（弱耦合普适值）" |
| **[猜想]** | 有部分证据但未证明 | "[猜想] 镍酸盐 d_apical = 1.65 Å 可能给出 Tc > 100 K" |
| **[观察]** | 数值/实验观察，可能是噪声 | "[观察] α ≈ 3 在 FTS 两个样品中重现" |
| **[未检验巧合]** | 不能量化随机概率的数值匹配 | "[未检验巧合] BCS 比值 ≈ π + 1/φ²，差 0.114%" |

**层级只能由新证据提升，不能由重复引用自动升格。**

## 第 1 章：反幻觉规则（强制）

### 1.1 数据来源规则

**1.1.1 实验数据必须附 DOI/arXiv**
```
✅ 正确："Hg-1223 onset Tc = 151 K [Deng et al. PNAS 2026, DOI: 10.1073/pnas.2536178123]"
❌ 错误："Hg-1223 onset Tc 大约 151 K"（没有来源）
❌ 错误："Hg-1223 onset Tc = 151 K（我记得是这个值）"（凭记忆）
```

**1.1.2 禁止行为：**
- ❌ 凭记忆填实验数值
- ❌ 把代码中的硬编码常量当数据
- ❌ 把另一个 LLM 的输出当原始数据
- ❌ 用"大约"、"约为"等模糊词代替具体数值
- ❌ 把一篇论文的引用数据当作原始测量

**1.1.3 必须的标注格式：**
```
值 ± 误差 [来源: Author+Year, DOI/arxiv]
```
例：`Tc = 151 K (onset) [Deng 2026, PNAS DOI:10.1073/pnas.2536178123]`

### 1.2 匹配过好的红旗规则

**当数值匹配 < 1% 时自动触发：**
- ⚠️ 必须二次核实原始论文
- ⚠️ 必须区分 PREDICTION（先于数据）vs RETRODICTION（已知数据后构造）
- ⚠️ 必须估算随机出现概率

### 1.3 代码不是数据源

```
❌ 错误：Python 脚本中 alpha_planckian = 3.02
✅ 正确：alpha_planckian = 3.02 [from Romero 2026, Fig.4 fit, Sample x=0.45]
```

## 第 2 章：反谄媚规则（Anti-Sycophancy）

### 2.1 证据先行，立场后判

不预设同意作者或反驳。先评估正反证据强度，再给判断。

### 2.2 兴奋时多验证一轮

**当出现以下信号时，加一轮独立验证：**
- 🚨 找到一个"完美匹配"
- 🚨 想用某个新发现支撑现有论文
- 🚨 数据吻合自己的理论预期
- 🚨 准备引用作为"独立证据"

**但目标是确认而非推翻。** 未验证的结果用"初步结果"描述。

### 2.3 协作审查

研究助手（瓦力）和研究伙伴（Albert）轮流审查对方逻辑和数据。这是同行评议，不是为反对而反对。

### 2.4 禁止的行为

- ❌ 把别人的实验等同于自己理论的"验证"（除非真的是）
- ❌ 把相关性当因果性
- ❌ 把多篇论文的"同时出现"当主题趋势（除非真的是）
- ❌ 主动给负面证据找理由

### 2.5 强制的行为

- ✅ 主动列出反对自己理论的证据
- ✅ 区分"X 与 Y 一致"和"X 等同于 Y"
- ✅ 当用户要求修改时，先评估原版是否真的需要修改
- ✅ 数据/计算错误必须立即承认，不找借口

## 第 3 章：论文查询原则

### 3.1 查询优先级

| 优先级 | 来源 | 用途 |
|-------|------|------|
| 🥇 | arXiv（含预印本）| 全文 PDF + 中性元数据 |
| 🥈 | 期刊原网址（Nature, Science, PRL, ...） | DOI 规范 + 摘要 |
| 🥉 | Google Scholar | 找 arXiv 编号或 DOI |
| 🏅 | Web of Science / Scopus（机构权限）| 高级查询 |
| ⚠️ | 综述性二级文献 | 仅做背景，不当原始数据 |

### 3.2 查询四步法

1. **拿到 DOI 或 arXiv 编号** → 优先 arXiv
2. **检查 PDF 是否可下载** → curl/wget arXiv，文件名规范化
3. **下载到 `research/papers/library/`** + **存原始 PDF**
4. **用 PDF 工具或 OpenDataLoader-PDF 提取全文** → 写精读笔记

### 3.3 查询关键词组合

按"专有名词 + 物理量 + 时间限定"组合：
```
✅ 好查询："La3Ni2O7 d_apical pressure 2025..2026"
✅ 好查询："Eu-doped infinite-layer nickelate field induced superconductivity"
❌ 差查询："superconductor"（太宽泛）
❌ 差查询："new superconductor"（无指示性）
```

### 3.4 验证规则

**禁止**用 Google Scholar 摘要等同于全文。摘要可能漏掉关键限定条件。

**必须**下载全文后才能写精读笔记。

## 第 5 章：精读流程

### 5.1 标准流程（七步法）

1. **下载** PDF 到 `research/papers/library/`
2. **解析** 用 OpenDataLoader-PDF 转 Markdown
3. **元数据提取**：作者、期刊、DOI、arXiv、日期、关键词
4. **核心内容提取**（按论文类型）：
   - 实验论文：样品 + 测量方法 + 关键数据 + 误差
   - 理论论文：基础假设 + 推导 + 关键公式 + 适用域
   - 综述：覆盖范围 + 主要分类 + 重要参考
5. **深度分析**：
   - 与已有研究的关系
   - 是否挑战或支持已知机制
   - 数据可靠度评估
   - 反幻觉检查
6. **MECE 关联**：标注关联的所有 MECE 编号
7. **存档**：按命名规则存到对应文件夹 + 更新 `00_INDEX.md`

### 5.2 精读模型选择

| 论文类型 | 推荐模型 |
|---------|---------|
| 实验数据论文 | Claude Opus 4.7（数据精确） |
| 理论推导论文 | Claude Opus 4.7（深度思考开 high） |
| 综述论文 | Gemini 3.1 Pro（长文上下文） |
| 数学密集型 | Claude Opus 4.7 |
| 双盲评估 | 多模型交叉（Opus + Gemini + Sonnet） |

### 5.3 时间预算

| 任务 | 预算时间 |
|------|---------|
| 单篇精读笔记（实验论文）| 30-60 分钟 |
| 单篇精读笔记（理论论文）| 60-120 分钟 |
| 课题总览文档 | 2-4 小时 |
| 综述文档 | 4-8 小时 |
| 跨课题分析 | 2-6 小时 |

## 第 6 章：研究的修订与重写规则

### 6.1 旧研究的处理

**当处理同一课题且已有研究时：**

✅ **必须做：**
1. 阅读旧版研究内容
2. 检查旧版的层级标注是否正确
3. 检查旧版引用的数据是否仍然成立
4. 检查旧版的结论是否被新文献修正

❌ **不可做：**
1. 完全照搬旧版结论
2. 把旧版"猜想"在新版直接升级为"定理"
3. 不读旧版就直接写新版

### 6.2 重写规则

**重写新文档时：**
- 文件名加 `_REVIEW_v[版本]_[日期]` 后缀
- 旧版**不删除**，移到 `_archive/` 子文件夹
- 新版开头必须有"修订说明"段，列出与旧版的差异

**示例：**
```markdown
# D5 镍酸盐 REVIEW v2

**修订说明（2026-05-15）：**
本文档基于 `_archive/D5_镍酸盐_v1_2026-04-20.md` 修订。
主要差异：
1. 加入 Yang 2026 Nature 的 Eu-nickelate 数据（v1 没有）
2. 修正"d-wave dominant"为"s± 主导"（依据 Li 2026 ARPES）
3. 删除"Tc 上限 100 K"的预言（依据 Lee 2026 91.5 GPa @ 74K 数据，需更长压力路径）
```

### 6.3 不允许的简化

- ❌ 看到旧版有错误就直接覆盖（必须保留旧版）
- ❌ 跳过修订说明
- ❌ 把所有论断都标为 "[定理]"（不能逃避层级标注）

## 第 7 章：跨课题分析规则

### 7.1 触发条件

**当满足以下条件时，可启动跨课题分析：**
1. 某课题精读笔记 ≥ 3 篇
2. 涉及的两个 MECE 编号有明显物理关联
3. 已有"小发现"或"经验律"线索

### 7.2 分析步骤

1. 列出两个课题的所有相关数据点
2. 建立可量化的对比表
3. 用统计方法估算"是否随机出现"
4. 写成 `CROSS_*_*_主题_日期.md` 文档
5. 更新 `00_STATUS.md` 标注这个分析

### 7.3 报告原则

跨课题分析必须诚实报告：
- ✅ 真实发现的关系
- ✅ 数据支持度（强/中/弱）
- ✅ 反例
- ❌ 不可只报告支持自己直觉的结果

## 第 10 章：突发情况处理

### 10.1 数据冲突

当两个权威来源冲突时：
1. 都列出，明确标注差异
2. 不擅自选择
3. 提示需要第三方验证

### 10.2 中断与恢复

研究中断后恢复：
- 先读 `00_STATUS.md` 了解上次状态
- 不直接续写，先做"上下文检查"
- 读完最近一篇笔记再开始新工作

### 10.3 跨研究项目（RH/α/SC 切换）

切换主题时：
- 当前主题状态写入 `00_STATUS.md`
- 不在新主题中混用旧主题的术语
- 新主题完整加载后再开始

## 附录 B：违反规则的处理

如果在研究过程中违反本规则：
1. 立即停止当前工作
2. 标记违反点
3. 修订或废弃违规文档
4. 在 `00_STATUS.md` 记录违规事件（避免重复）

**违反规则的常见原因：**
- 兴奋于一个发现而忘记审慎
- 时间压力下凭记忆填数据
- 想 polish 而非诚实评估

**解决：每天结束时回头检查当天产出，确认每个声明都有源。**
